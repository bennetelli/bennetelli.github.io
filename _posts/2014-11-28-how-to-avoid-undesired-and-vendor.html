Enterprise projects typically have a large amount of different dependencies, libraries and other stuff. This count of classes increases when using IBM JDK with its own implementations of different Oracle JDK classes.
In case you are working on a large enterprise project you possibly know this nuisances.

As you can see in the following figure you even get three different suggestions when typing "Logger" using Oracle's JDK without any dependencies and libraries.



This count of suggestions increases rapidely when using IBM J9 or different dependencies and libraries. Sometimes it appears that someone chooses the wrong import. For example cmo.ibm.icu.util.Calendar instead of java.util.Calendar. Luckily and/or unluckily both Calendar implementations have equal methods and also method signatures.

This results in an undesired import and eliminates you're additionally married with the IBM JDK! Congratulations! To avoid undesired suggestions and imports, Eclipse offers a feature named type filters. You simply have to choose:  

Window -> Preferences -> Java -> Appearances -> Type Filters

A click on Add Packages... and you get the following filter window.


There you can add specific classes or whole packages. After that Eclipse won't suggest these packages anymore.

It is also possible to hide methods like wait(), notifyAll() and other outdated methods like the awt and swing ones. 

Have fun with this more clear and stress-free programming :-)
